Q1

IAM Users vs Roles?

  • Users: Long-term credentials, for people, password/access keys
  • Roles: Temporary credentials (STS), for services/cross-account/federation
Q2

SCP vs IAM Policy?

SCP: Organization-level guardrails, doesn't grant permissions, restricts entire account including root.

IAM Policy: Grants permissions to user/group/role, doesn't affect root.

Q3

Secrets Manager vs Parameter Store?

  • Secrets Manager: Auto-rotation, RDS integration, $0.40/secret/month
  • Parameter Store: Free tier, hierarchical, good for config values
Q4

KMS Envelope Encryption?

KMS API only encrypts up to 4KB. For larger data: generate Data Key (DEK), encrypt data locally with DEK, store encrypted DEK + encrypted data. Decrypt by first decrypting DEK with KMS.

Q5

WAF vs Shield?

  • WAF: Layer 7, SQL injection, XSS, IP blocking, rate limiting
  • Shield: DDoS protection, Standard (free) for common attacks, Advanced for sophisticated

Q7

Cross-account access flow?

1. Create role in target account with trust policy allowing source account. 2. Source account user/role calls sts:AssumeRole. 3. STS returns temporary credentials. 4. Use temp creds to access target account resources.

Q8

CloudTrail vs Config?

  • CloudTrail: API audit log, who did what when
  • Config: Resource configuration tracking, compliance rules, what changed
