Custom Metrics

Important
JavaPublishing Custom Metrics
CloudWatchClient cloudWatch = CloudWatchClient.create();

// Put custom metric
PutMetricDataRequest request = PutMetricDataRequest.builder()
    .namespace("MyApp/Orders")
    .metricData(MetricDatum.builder()
        .metricName("OrdersProcessed")
        .value(42.0)
        .unit(StandardUnit.COUNT)
        .dimensions(
            Dimension.builder().name("Environment").value("Prod").build(),
            Dimension.builder().name("Region").value("us-east-1").build()
        )
        .storageResolution(1) // High resolution (1 second)
        .build())
    .build();

cloudWatch.putMetricData(request);

Custom Metrics Notes

  • Standard resolution: 1-minute minimum (free)
  • High resolution: 1-second ($0.30 per metric/month)
  • Can use PutMetricData API or CloudWatch Agent

-- Top 20 most recent errors
fields @timestamp, @message
| filter @message like /ERROR/
| sort @timestamp desc
| limit 20

-- Latency percentiles by API
fields @timestamp, @message
| parse @message /Duration: (?[\d.]+) ms/
| stats avg(duration) as avg_ms, 
        pct(duration, 95) as p95_ms,
        pct(duration, 99) as p99_ms
  by bin(1h)

-- Count errors per Lambda function
fields @timestamp, @message, @logStream
| filter @message like /ERROR/
| stats count() as error_count by @logStream
| sort error_count desc
