Feature Redis Memcached

FlowCaching Strategies
Cache-Aside (Lazy Loading):
1. App checks cache
2. Cache miss â†’ Query DB â†’ Write to cache
3. Next request hits cache
Pro: Only cache what's needed
Con: Cache miss penalty, stale data

Write-Through:
1. App writes to cache AND DB
2. Cache always up-to-date
Pro: Fresh data
Con: Write penalty, unused data cached

TTL (Time-to-Live):
- Balance freshness vs performance
- Typical: 15 min - 24 hours
- Critical data: shorter TTL
