Essential
"Open for extension, closed for modification."

L - Liskov Substitution Principle

Essential
"Subclasses must be substitutable for their base classes."

Classic Violation: Rectangle-Square

Square.setWidth() changes both width AND height - breaks Rectangle's expected behavior!

LSP Rules

  • Preconditions cannot be strengthened in subclass
  • Postconditions cannot be weakened
  • Invariants must be preserved

Q1

Abstraction vs Encapsulation?

Encapsulation: HOW to hide (access modifiers, getters/setters). Abstraction: WHAT to hide (interface, abstract class).

Q2

Why doesn't Java support multiple inheritance?

To avoid Diamond Problem. If A extends B and C, and both have method foo(), which one to inherit? Java uses interfaces instead.

Q3

Can static methods be overridden?

No. They can only be hidden. Static methods use static binding (compile-time), not dynamic binding.

Q4

When to use Abstract Class vs Interface?

Abstract Class: IS-A + shared code. Interface: CAN-DO capability, multiple inheritance of type.
