Object Lifecycle in Heap

  • 1. New object created in Eden space
  • 2. Minor GC: Survivors move to S0 or S1 (alternating)
  • 3. After surviving multiple Minor GCs (threshold), move to Old Gen
  • 4. Major GC cleans Old Gen when full

Class Loading Process

  • Loading: Read .class file, create Class object
  • Linking:
    • Verification: Verify bytecode
    • Preparation: Allocate memory for static vars (default values)
    • Resolution: Replace symbolic refs with direct refs
  • Initialization: Execute static blocks, initialize static vars

Delegation Model

ClassLoader asks parent first before loading itself (parent-first delegation). This ensures core Java classes are loaded by Bootstrap ClassLoader.

G1 GC (Default since Java 9)

  • Divides heap into equal-sized regions
  • Prioritizes regions with most garbage (Garbage First)
  • Concurrent marking, incremental compaction
  • Target pause time: -XX:MaxGCPauseMillis=200

  • Tiered Compilation: Combines both - interpret first, compile when hot

Q1

Explain JVM memory model?

Heap: Objects (shared). Stack: Local vars, method calls (per thread). Metaspace: Class metadata. PC Register: Current instruction.

Q2

How does G1 GC work?

Divides heap into regions. Collects regions with most garbage first. Uses concurrent marking. Target pause time with MaxGCPauseMillis.

Q3

What causes memory leak in Java?

Objects still referenced but not needed: Static collections, unclosed resources, listeners not removed, ThreadLocal not cleaned, inner class holding outer reference.

Q4

What is ClassLoader delegation model?

Parent-first delegation: Child asks parent to load first. Prevents loading core classes from untrusted sources. Bootstrap â†’ Extension â†’ Application â†’ Custom.

Q5

When is an object eligible for GC?

When there are no reachable references from GC roots. Setting reference to null, reassigning, going out of scope. finalize() is deprecated - use try-with-resources.
